# Red Ball Finance – Paper Portfolio

Public, transparent paper portfolio (in ₹) with a detailed write-up for every buy,
benchmarked against the Nifty 50.

## Dev
No build step. Just static files — open `index.html` (or serve the folder).

- `data/transactions.json` is the **source of truth** for what you bought.
- `data/stocks.json` holds display metadata (name, tags, thesis, `slug`).
- Each holding's `slug` maps to `reports/<slug>.md`.
- Everything else in `data/` is generated by the GitHub Action.

## How the data pipeline works
`scripts/ci/build_data.py` runs daily (and on demand) and rebuilds, all in ₹:

| File | Contents |
|------|----------|
| `data/quotes/<SYM>.json` | per-stock daily closes |
| `data/quotes/NIFTY50.json` | Nifty 50 benchmark closes |
| `data/dividends.json` | dividend events + total ₹ received |
| `data/positions.json` | per-holding qty, cost, value, dividends, P&L |
| `data/nav.json` | daily NAV series + `nifty_index` benchmark |
| `data/nav_summary.json` | latest snapshot for the cards |

Price source order per symbol: **Yahoo Finance** (prices + dividends, no API key)
→ **Google Sheet** tab (optional fallback, set repo secret `SHEET_ID`)
→ existing cached quote file (so a failed fetch never wipes data).

## Add / change a holding
1. Edit `data/transactions.json` — add `{ "symbol": "TICKER.NS", "date": "YYYY-MM-DD", "price_local": <buy price ₹>, "amount_inr": <₹ invested> }`.
2. Add a matching entry in `data/stocks.json` (same `symbol`, plus `slug`, `name`, `tags`).
3. Create `reports/<slug>.md`.
4. Commit & push. The Action rebuilds the numbers; GitHub Pages auto-deploys.

To change the starting capital, set the `STARTING_CASH` env in
`.github/workflows/fetch-prices.yml` (default `10000000` = ₹1 crore).
